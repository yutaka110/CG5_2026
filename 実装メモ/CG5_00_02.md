# Grayscale PostProcess 実装メモ

## 概要

授業資料の Grayscale 処理を、このプロジェクトの既存構成に合わせて
`PostProcessStack` の 1 パスとして実装した。

資料では `CopyImage` 系の Fullscreen Shader を複製して Grayscale 化する流れだったが、
本プロジェクトにはすでに以下の仕組みがある。

- `FullscreenComposite.VS.hlsl` による fullscreen triangle 描画
- `PostProcessStack` によるポストプロセスパス管理
- `AppPostProcessPipeline` による RenderGraph へのパス登録
- `AppPipelines` による PSO / Shader 管理
- ImGui の PostProcess パネル

そのため、`CopyImage` を新規作成するのではなく、
既存のポストプロセス基盤に `Grayscale` パスを追加する方針にした。

### `application/AppPostProcessPanel.cpp`

ImGui 上で Grayscale を操作できるようにした。

既存の共通 UI により、以下は自動で表示される。

- `Grayscale` の ON / OFF チェックボックス
- `Intensity` スライダー

## ImGui での確認方法

1. アプリを起動する。
2. VFX Engine / PostProcess パネルを開く。
3. `Grayscale` のチェックボックスで ON / OFF を切り替える。
4. `Intensity` を操作する。
   - `0.0`: 元のカラー
   - `1.0`: 完全な Grayscale
5. `Mode` で変換方式を切り替える。
   - `Average`: RGB の単純平均
   - `BT.709`: 人間の明るさ知覚に寄せた係数
