# Outline PostProcess 実装メモ

## 概要

授業資料の Outline / Edge Detection 処理を、このプロジェクトの既存構成に合わせて
`PostProcessStack` の 1 パスとして実装した。

資料では輝度差分を使った PrewittFilter をもとにエッジを検出する流れだったが、
VFX エンジンとして使いやすい Outline にするため、今回は以下の情報を組み合わせる方針にした。

- 輝度ベースの Prewitt edge
- `SceneDepth` による depth edge
- `SceneNormalObjectId.rgb` による normal edge
- `SceneNormalObjectId.a` による object mask / object id edge

本プロジェクトにはすでに以下の仕組みがある。

- `FullscreenComposite.VS.hlsl` による fullscreen triangle 描画
- `PostProcessStack` によるポストプロセスパス管理
- `AppPostProcessPipeline` による RenderGraph へのパス登録
- `AppPipelines` による PSO / Shader 管理
- ImGui の PostProcess パネル
- `SceneDepth` の SRV
- RenderGraph による render target / depth target の管理

そのため、`CopyImage` 系の Shader を新規作成するのではなく、
既存のポストプロセス基盤に `PrewittOutline` パスを追加する方針にした。

また、VFX エンジン向けの拡張として、
3D モデル描画時に `SceneNormalObjectId` という MRT を追加した。

- `rgb`: world normal を `normal * 0.5 + 0.5` で 0.0 - 1.0 にエンコード
- `a`: object mask / object id 用の値

今回は main model を `1.0`、背景を clear color の `0.0` として扱う。
これにより、色だけでは検出しづらい輪郭も depth / normal / object mask で補えるようにした。

`SceneNormalObjectId` の追加により render target が増えたため、
`DescriptorHeap exhausted` を避けるために descriptor heap の容量も拡張した。

- RTV: `8` から `64`
- DSV: `8` から `16`
- SRV: `1024` から `2048`

## ImGui での確認方法

1. アプリを起動する。
2. VFX Engine / PostProcess パネルを開く。
3. `PrewittOutline` のチェックボックスで ON / OFF を切り替える。
4. `Intensity` を操作する。
   - `0.0`: Outline なし
   - `1.0`: Outline を通常適用
   - `1.0` 以上: Outline をより強く表示
5. `Threshold` を操作する。
   - 値を小さくすると細かい差分も Outline として拾いやすい
   - 値を大きくすると強い境界だけを Outline として拾う
6. `Thickness` を操作する。
   - `1.0`: 1 texel 周辺を参照
   - 値を大きくすると太めの輪郭になる
7. `Softness` を操作する。
   - 値を小さくすると硬い輪郭になる
   - 値を大きくすると輪郭の出方がなめらかになる
8. `Depth Weight` を操作する。
   - depth 差分による輪郭の強さを調整する
   - 形状の前後関係やシルエットを拾いやすくなる
9. `Normal Weight` を操作する。
   - normal 差分による輪郭の強さを調整する
   - 同じ色や近い depth でも面の向きが変わる部分を拾いやすくなる
10. `Object Weight` を操作する。
    - object mask / object id 差分による輪郭の強さを調整する
    - 背景と main model の境界を拾いやすくなる
11. `Outline Color` で輪郭色を変更する。
    - 黒: アニメ調の輪郭
    - 明るい色: VFX 的な発光輪郭や強調表現に使いやすい
