# Vignette PostProcess 実装メモ

## 概要

授業資料の Vignetting 処理を、このプロジェクトの既存構成に合わせて
`PostProcessStack` の 1 パスとして実装した。

資料では `CopyImage` 系の Fullscreen Shader を基に `Vignette.PS.hlsl` を作る流れだったが、
本プロジェクトにはすでに以下の仕組みがある。

- `FullscreenComposite.VS.hlsl` による fullscreen triangle 描画
- `PostProcessStack` によるポストプロセスパス管理
- `AppPostProcessPipeline` による RenderGraph へのパス登録
- `AppPipelines` による PSO / Shader 管理
- ImGui の PostProcess パネル

そのため、`CopyImage` を新規作成するのではなく、
既存のポストプロセス基盤に `Vignette` パスを追加する方針にした。

Vignette は画面端を暗くする最終画面演出なので、
Particle / Trail / Beam / Distortion のような VFX Technique としては扱わず、
`TechniqueRegistry` には登録していない。

また、`Grayscale` と `Vignette` の ON / OFF を個別に切り替えても正しい直前出力を読めるように、
`PostProcessOutput` と `PostProcessSwapOutput` を追加した。
これにより、最終調整系のポストエフェクトは直前の出力を読み、
読み込み先と重ならない一時 RenderTarget に書き込む。

## ImGui での確認方法

1. アプリを起動する。
2. VFX Engine / PostProcess パネルを開く。
3. `Vignette` のチェックボックスで ON / OFF を切り替える。
4. `Intensity` を操作する。
   - `0.0`: 元の明るさ
   - `1.0`: Vignette を最大適用
5. `Radius` を操作する。
   - 値を小さくすると暗くなる範囲が中心側に広がる。
   - 値を大きくすると暗くなる範囲が画面端側に寄る。
6. `Softness` を操作する。
   - 値を小さくすると境界がはっきりする。
   - 値を大きくするとなだらかに暗くなる。
7. `Power` を操作する。
   - 値を上げると減衰カーブが強くなる。
   - 値を下げると自然で薄い効果になる。
