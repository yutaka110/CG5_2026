# OffscreenRendering / RenderTexture 課題メモ

## 概要

授業資料では、RenderTextureを作成し、そこにSceneを描画したあと、
専用のCopyImageシェーダを使ってSwapChainへコピーする流れが説明されている。

本プロジェクトでは、資料と同等の処理を以下の構成で実装している。

## 資料との対応

| 授業資料の内容 | 本プロジェクトでの実装 |
|---|---|
| RenderTexture Resource の作成 | `AppVfxRenderTargets::CreateTarget()` |
| `D3D12_RESOURCE_FLAG_ALLOW_RENDER_TARGET` の設定 | `AppVfxRenderTargets::CreateTarget()` |
| ClearValue の設定 | `AppVfxRenderTargets::CreateTarget()` |
| RTV の作成 | `AppVfxRenderTargets::CreateTarget()` |
| SRV の作成 | `AppVfxRenderTargets::CreateTarget()` |
| SceneをRenderTextureへ描画 | `AppSceneRenderPipeline.cpp` の `SceneColor` |
| RenderTextureをSRVとして読む | `AppPostProcessPipeline.cpp` |
| SwapChain / BackBufferへ出力 | `AppVfxRenderTargets::CompositeToBackBuffer()` |
| CopyImage.VS.hlsl 相当 | `FullscreenComposite.VS.hlsl` |
| CopyImage.PS.hlsl 相当 | `FullscreenComposite.PS.hlsl` |
| `SV_VertexID` によるFullscreen三角形 | `FullscreenComposite.VS.hlsl` |
| `DrawInstanced(3, 1, 0, 0)` | `AppVfxRenderTargets::CompositeToBackBuffer()` |
| ResourceBarrier | `RenderGraph.cpp` / `AppVfxRenderTargets::Transition()` |

## 実装方針

授業資料では `CopyImage.VS.hlsl`、`CopyImage.PS.hlsl`、`CopyImage.hlsli` を作成しているが、
本プロジェクトでは同じ役割の処理を `FullscreenComposite` として実装している。

`FullscreenComposite.VS.hlsl` では、頂点バッファを使わず、
`SV_VertexID` によって全画面三角形を生成している。

`FullscreenComposite.PS.hlsl` では、RenderTextureとして描画された `SceneColor` などを
SRVとしてSampleし、最終的にBackBufferへ出力している。

## 描画の流れ

Scene / Model / Sprite
↓
SceneColor RenderTexture に描画
↓
SceneColor を SRV として読み取り
↓
FullscreenComposite Shader で BackBuffer に描画
↓
SwapChain Present
